# 风险管理驾驶舱项目最高优先级原则

最高优先级原则：
本项目当前使用虚拟数据开发，但后续必须能够被 IT 同事快速理解、快速替换为真实数据并快速上线，因此所有开发都必须以“开发逻辑简洁、页面内容简洁、非必要不增加”为第一原则。

执行口径：
1. 能少一层逻辑就少一层逻辑，避免不必要的状态、联动、嵌套和特殊分支。
2. 能少一个元素就少一个元素，避免不必要的说明文字、装饰内容、冗余标签和重复交互。
3. 所有筛选、图表、表格和切换逻辑都优先采用直观、可替换、易交接的实现方式。
4. 虚拟数据的组织方式要尽量贴近未来真实数据替换方式，避免为了演示效果引入复杂的临时结构。
5. 当“展示效果更花哨”和“后续接手更容易”发生冲突时，一律优先后者。

后续开发判断标准：
如果某项设计、交互或代码不能明显帮助 IT 同事更快理解、更快替换数据、更快上线，就默认不做。
