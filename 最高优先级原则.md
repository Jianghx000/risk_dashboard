# 风险管理驾驶舱项目最高优先级原则

本项目当前是可汇报、可演示的前端原型，但后续需要能被 IT 同事快速理解、快速替换为真实数据并接入正式系统。因此最高优先级是：

**结构清楚、逻辑简洁、数据口径集中、页面不过度堆叠。**

执行口径：

1. 能少一层结构就少一层结构，不为了视觉层级保留旧的数据层级。
2. 能用配置表达的业务口径，不写进 `app.js`。
3. 能在 `dashboard-data.js` 删除的废弃图表，不靠前端隐藏。
4. 能用 `dashboard-domain.js` 统一维护的业务分类、业务分组、套期项目，不分散在多个文件。
5. 能用 `validate_dashboard.py` 检查的规则，不依赖人工记忆。
6. 展示效果和后续接手便利冲突时，优先后续接手便利。

当前事实源：

- 页面结构：`prototype/dashboard-data.js`
- 页面配置：`prototype/dashboard-config.js`
- 领域口径：`prototype/dashboard-domain.js`
- 共享工具：`prototype/dashboard-utils.js`
- 页面渲染骨架：`prototype/app.js`
- 模拟测算：`prototype/dashboard-simulation.js`
- 计算过程拆解：`prototype/dashboard-processes.js`
- 利率风险渲染：`prototype/dashboard-interest-renderers.js`
- 业务变动渲染：`prototype/dashboard-business-renderers.js`
- 流动性与投融资渲染：`prototype/dashboard-liquidity-renderers.js`
- 渲染注册：`prototype/dashboard-renderers.js`
- DOM 事件绑定：`prototype/dashboard-events.js`
- 样式：`prototype/styles.css`

后续开发判断标准：

如果某项设计、交互或代码不能明显帮助用户理解风险，也不能帮助后续真实数据接入，就默认不做。
